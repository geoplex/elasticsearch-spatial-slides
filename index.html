What is Elasticsearch?

An Open Source, Distributed, RESTful Search Engine

or...

A fancy wrapper around Lucene

Some Highlights

  1. Unstructured and Structured Search
  2. Distributed & Highly Available
  3. Well documented Restful API
  4. Analysis i.e. aggregates
  5. .....and ofcourse it has Geospatial goodies!

What is Elasticsearch....not

  1. A GIS
  2. Something to make maps with
  3. A persistant store
  4. The answer to all your problems

Why bother?

  1. When you want to deal with alot of 'event' data
  2. When you want to have a better search experience
  3. When you want to service alot of demand
  4. When you want to deal with unstructured data
  5. Because searching for stuff on maps often sucks!

Getting Spatial Data In

  1. Indexes - think of it as a database
  2. Documents - are stored in indexes and are modelled as JSON
  3. Types - Indexes can contain types of documents - think of these as tables
  4. Mapping - define the schema of a type - automatically inferred by Elasticsearch
  5. Types - a datatype for the fields in documents

1. Create Index


$ curl -XPUT 'http://localhost:9200/accidents/'

2. Insert Mapping


							curl -s -XGET "http://127.0.0.1:9200/accidents/accident/_mapping?pretty=true" > mapping.json

By default Elasticsearch does not automatically create the correct field types for Geospatial data, therefore we have to explicitly create a mapping which includes the correct Geospatial type.
